AgentFlow Roadmap v0.1

2026-05-26 · #️⃣ e1bd29a

Turn prompts into pipelines.

AgentFlow is an open-source orchestration ecosystem for LLM workflows. Local-first, security-hardened, provider-agnostic. This roadmap shows every component, where it lives, and what's coming next.

core principle Raw session truth stays local first. Cloud gets summarized, intentional sync data.

Ecosystem Map

agentflow-engine live
Data bus. Parses, analyzes, stores session data from every producer. Serves local HTTP API on :8377. Python stdlib + SQLite.
Python 3.11 · SQLite · stdlib http.server
agentflow-ai-chat-deck live
Session producer. Dual-model adversarial conversation deck. Two AI models conversing automatically with user injection controls. POSTs completed sessions to engine.
Single HTML · JS · localStorage · CORS relay
agentflow-vscode upcoming
Metrics consumer. Workbench dashboard + architect chat panels inside VS Code. Reads from engine HTTP API with fallback chain.
TypeScript · VS Code API · Webviews
agentflow-site live
Marketing site + ecosystem hub. Product pages, docs, and navigation to all ecosystem surfaces. Hosted on CF Pages.
HTML · CSS · JS · Cloudflare Pages
sync-worker future
Cloud sync layer. Receives batched metric summaries from engine, stores in D1 for the cloud dashboard. Optional — doesn't block local dev.
Cloudflare Workers · D1 · API
agentflow (web app) future
Cloud dashboard. Reads synced metric summaries from D1. Team-wide visibility without raw session data leaving local.

Build Sequence

#1

Engine HTTP API done

Stdlib HTTP server on :8377 wrapping the analysis pipeline. POST /v1/ingest/session, GET /v1/health, GET /v1/metrics/summary, GET /v1/sessions. Contract converter normalizes external payloads. Idempotent ingestion. 20/20 tests passing.

#2

Chat Deck → Engine Wiring done

On session complete/stop, transforms deck session to engine contract and POSTs to localhost:8377. If engine is down, queues in localStorage with 30s retry interval. Fire-and-forget — no UI impact.

#3a

Ecosystem Mockups done

Control Plane dashboard, Workbench page, Chat Deck landing page, shared Drift v2 CSS. Pushed to agentflow-site main. Nav updated across all pages. Awaiting Jules design review.

#3b

Design Review in progress

Jules (session 9335634489885989930) reviewing mockups against design brief. Checking brand consistency, responsiveness, component completeness. Will return structured feedback as PR.

#4

VS Code Extension ← Engine next

Update MetricsProvider to try HTTP engine first, fall back to JSON/SQLite. Add workbench webview panels for prompt analysis and session explorer.

#5

Sync Worker future

Thin CF Worker that receives batched summaries from engine, writes to D1. Enables cloud dashboard without raw session data leaving local.

#6

Web App → D1 future

Cloud dashboard views that read from sync Worker API instead of mock data. Team-wide visibility.
