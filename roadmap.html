Build Sequence

#0

Engine HTTP API done

Stdlib HTTP server on :8377. POST /v1/ingest/session, GET /v1/health, GET /v1/metrics/summary, GET /v1/sessions. Contract normalizer. Idempotent ingestion. 20/20 tests.

#0b

Chat Deck → Engine Wiring done

POST on session save, localStorage retry queue, 30s periodic retry. Fire-and-forget — no UI impact. Ready for re-purposing to agent refinement.

#1

Architect Chat top priority

Primary user interaction. Onboard users, generate the one consistent agent template. Produces data sets, skills, config, agent criteria. Spawns AGENTS.md subagent (CF Worker alpha) → git commit. This is where users start every workflow.

#2

AGENTS.md Subagent (Alpha Flow) next

CF Worker (or injected sub-agent) that generates AGENTS.md from the Architect Chat template. Commits to git alongside config, skills, and agent data sets. The first automated output of the template pipeline.

#3

Chat Deck — Agent Refinement Mode next

Re-purpose Chat Deck from dual-model chat to agent definition decomposition. Takes template from Architect Chat, refines criteria through adversarial conversation. Produces orchestration loops: plan → execute → review. Generates initial agent criteria for sub-agents.

#4

Library Page future

Browse and deploy gateway. All generated agents, definitions, criteria. Push to any harness: Codex CLI, agy CLI, Claude, GitHub Copilot, Azure Copilot Studio, or the engine. Inject criteria as context.

#5

Harness Integrations future

Per-harness push adapters. Codex gets system prompt injection. agy gets context packet. Azure CS gets API config export. Each harness receives the same agent criteria, adapted to its interface.

#6

Sync Worker + Cloud future

Batched metric summaries from engine → CF Worker → D1. Cloud dashboard for team-wide trends. No raw session data leaves local.
