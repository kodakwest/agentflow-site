01 — Foundation

Design Philosophy

Four principles govern every decision in the Drift system. They are not aspirational — they are constraints. If a design doesn't satisfy all four, it doesn't ship.

Principle 01
State Visibility
Token counts, turn counters, status indicators shown inline, never hidden. The user should always know what the system is doing without opening a drawer or hovering for a tooltip.
Principle 02
Dual-Source Clarity
Model A (teal) and Model B (violet) are visually distinct at a glance. Every piece of AI-generated content is source-attributed through color, never ambiguous.
Principle 03
Controls Over Automation
Start/Pause/Stop/Retry always accessible, never trapped in menus. Users must feel in command of the conversation loop at all times — automation serves them, not the reverse.
Principle 04
Debug Transparency
Raw API payloads, token flow, latency shown in terminal, not hidden behind settings. Power users should be able to inspect the full data path without leaving the UI.
Design DNA
Retrofuturist Functional Minimal Alive

Drift looks like a starship control panel. Feels like floating through deep space. Works like a precision instrument. Every element earns its place on screen.

✓ Do

Use JetBrains Mono for status readouts, uppercase labels, terminal elements, section headings, and tags.

Use Inter for all body text, buttons, and form labels.

Reserve Playfair Display exclusively for brand hero titles.

✗ Don't

Don't use Playfair Display for body text — it lacks the x-height for comfortable reading.

Don't use Inter for code blocks or terminal output.

Don't mix font families within the same text block.

06 — Glass

Glassmorphism Patterns

Glass layers add depth without obscuring the dark space aesthetic. The recipe is simple: semi-transparent background, backdrop blur, and a subtle teal border.

Glass Card Recipe

CSS 
.glass-element {
  background: var(--glass-bg);           /* rgba(17, 18, 21, 0.7) */
  backdrop-filter: blur(var(--glass-blur)); /* 16px */
  -webkit-backdrop-filter: blur(var(--glass-blur));
  border: 1px solid var(--glass-border);  /* rgba(34, 211, 238, 0.15) */
  border-radius: 8px;
}

✓ Do

Use glass effects over complex or dark backgrounds to create depth separation.

Keep blur at 16px for consistency across all glass elements.

Apply the teal glass-border for thematic cohesion.

✗ Don't

Don't stack multiple glass layers — blur compounds and creates visual mud.

Don't use glassmorphism on light backgrounds — the effect requires contrast.

Don't increase glass-bg opacity above 0.8 — it defeats the purpose.

Scroll Reveal Pattern

CSS + JS 
/* CSS */
.reveal {
  opacity: 0;
  transform: translateY(16px);
  transition: opacity 0.5s var(--ease-out), transform 0.5s var(--ease-out);
}
.reveal.revealed {
  opacity: 1;
  transform: translateY(0);
}

/* JS — IntersectionObserver */
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('revealed');
      observer.unobserve(entry.target);
    }
  });
}, { threshold: 0.1 });

document.querySelectorAll('.reveal').forEach(el => observer.observe(el));

09 — Layout

Responsive Breakpoints

Two breakpoints handle the transition from desktop to mobile. The design maintains all functionality at every width — nothing is removed, only reflowed.

BreakpointWidthChanges
Tablet≤ 980pxCollapse to 2 columns · Hide history panel · Stack grids · Sidebar becomes slide-out drawer
Mobile≤ 640pxSingle column · Stack buttons vertically · Reduce padding · Simplify grid layouts

Reduced Motion

CSS 
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
  .reveal { opacity: 1; transform: none; }
  html { scroll-behavior: auto; }
}

Keyboard & ARIA

  • • All interactive elements are reachable via Tab key navigation
  • • Custom toggles use native <input type="checkbox"> for screen reader compatibility
  • • Navigation landmarks use role="navigation" and aria-label
  • • Status dots use adjacent text labels — never color alone to convey state
  • • All images and icons include descriptive alt text or aria-label
  • • Modal dialogs trap focus and return it on close
